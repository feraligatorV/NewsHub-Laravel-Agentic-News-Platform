Authentication validation rules:

- The project must use JWT authentication for API access.
- The project must use tymon/jwt-auth.
- Protected API routes must require Authorization: Bearer <token>.
- Do not replace JWT with Sanctum.
- If Sanctum appears only because of Breeze/Inertia scaffolding, document it as non-primary and do not use it for API authentication.

Testing tasks:

- Create or improve tests for:
  - User registration.
  - User login.
  - Login returns JWT token.
  - Protected route rejects unauthenticated requests.
  - Authenticated user can access /api/auth/me.
  - News listing.
  - News detail.
  - Recommended news.
  - Categories.
- Run the backend test suite.
- Run the frontend production build.
- Run Docusaurus build if possible.

Expected commands:

- From backend:
  - php artisan test
  - npm run build

- From docs-site:
  - npm run build

Because PHP, Composer and Node may not be installed on the host machine, prefer Docker commands when necessary.

Use examples:

- docker run --rm -v "${PWD}:/app" -w /app composer:2 php artisan test
- docker run --rm -v "${PWD}:/app" -w /app node:22 npm run build

Fix failing tests only if the fix is clearly related to the tested behavior.
Do not change backend or frontend business logic unless needed to fix a verified defect.
Do not introduce new features.
